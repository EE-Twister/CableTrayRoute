Reliability & N-1 Contingency Analysis

Method & Assumptions

Component availability is computed from Mean Time Between Failures (MTBF) and Mean Time To Repair (MTTR) values per IEEE Std 493 (Gold Book) and IEEE Std 1366:

A = MTBF / (MTBF + MTTR)

Expected outage hours per year per component:

downtime = (8760 / MTBF) × MTTR

System unavailability is computed using minimal cut sets up to N-2 failure combinations. Components are loaded from the active One-Line Diagram. Set MTBF and MTTR values on each component in the One-Line editor to enable analysis.

Connector elements (cables, feeders, links) are excluded from the component count — they contribute to path unavailability rather than as standalone components.

References: IEEE Std 493-2007, IEEE Std 1366-2012, IEC 60300-3-2.

Reliability Analysis Help

This tool computes system reliability metrics from your One-Line Diagram component data.

Requirements: Components in the One-Line Diagram must have MTBF and MTTR values set (in hours). Edit component properties in the One-Line editor.

Override values: After loading, you can adjust MTBF/MTTR for any component and re-run the analysis without modifying the stored diagram data.

N-1 analysis: Shows which single component failure causes the greatest system unavailability impact.
