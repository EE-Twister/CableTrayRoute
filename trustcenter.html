Trust Center

Verify that CableTrayRoute calculation engines produce results that match published industry benchmarks derived from IEEE, IEC, NEC, and NFPA standards. Each benchmark is a canonical known-answer problem with analytically-derived expected values.

About These Benchmarks

Each benchmark specifies fixed inputs and an expected output derived analytically from a first-principles formula or a table in the governing standard — not from the code under test. A passing result means the engine matches the standard, not just that it is internally consistent.

Tolerances account for IEEE 754 floating-point rounding. Results outside tolerance indicate a potential regression in the calculation engine and should be investigated before relying on study outputs for design decisions.

Benchmarks currently cover: EMF Analysis (Biot-Savart law, ICNIRP 2010), Battery / UPS Sizing (IEEE 485-2010), and Voltage Drop (NEC 2023 Art. 210.19/215.2 informational-note recommendations).

Trust Center Help

The Trust Center runs a suite of known-answer benchmarks against the calculation engines used in CableTrayRoute studies.

  • Run All Benchmarks — executes every benchmark and shows pass/fail status for each check.
  • Details — click any row to expand the individual checks, showing expected vs. actual values and % deviation.
  • Green ✓ — the engine output matches the analytically-derived expected value within tolerance.
  • Red ✗ — the output deviates beyond the allowed tolerance. Check the engine for regressions.

Benchmarks are re-run in-browser against the live code — no server request is made. Results are not stored and do not affect project data.
