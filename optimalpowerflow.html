Optimal Power Flow / Economic Dispatch

Method & Assumptions

Economic dispatch finds the generation schedule that meets system demand at least total fuel cost, subject to each unit's minimum and maximum output limits. Each unit's fuel cost is a convex quadratic curve, and the optimum is found by the classic equal-incremental-cost (lambda) condition.

Cost model:

C_i(P) = a_i + b_i·P + c_i·P²      [$/h]
IC_i(P) = b_i + 2·c_i·P            [$/MWh]

At the optimum, every unit operating between its limits runs at the same system incremental cost λ. Units whose incremental cost would exceed λ at Pmin are held at Pmin; units that reach λ only beyond Pmax are capped at Pmax. The system λ that satisfies Σ Pi = demand + losses is found by bisection.

Losses: modelled at screening level as a flat percentage of demand. Required generation = demand × (1 + loss% / 100).

Savings: total cost is compared against a naive capacity-proportional dispatch to quantify the value of cost-aware scheduling.

References: Wood, Wollenberg & Sheblé, Power Generation, Operation, and Control, 3rd ed., §3; IEEE Std 399-1997 §3.

Optimal Power Flow / Economic Dispatch Help

This study schedules a fleet of dispatchable generators to meet a system demand at the lowest total fuel cost, respecting each unit's output limits.

When to use

  • Microgrid and on-site generation planning — how should engines/turbines share a load to minimise fuel cost?
  • Multi-generator standby plants — find the most economical loading split.
  • Estimating the marginal (incremental) cost of energy for a given demand.
  • Quantifying the savings of cost-aware dispatch versus running every unit proportionally.

Entering generators

  • Pmin / Pmax: the stable operating range of the unit in MW.
  • a, b, c: fuel-cost curve coefficients for C = a + b·P + c·P² ($/h). b is the linear ($/MWh) term and c the quadratic ($/MWh²) term; c must be ≥ 0 (convex).
  • Use Load Demonstration Fleet for the textbook 3-unit example (850 MW → λ ≈ 9.15 $/MWh, ≈ $8,194/h).
  • Import a CSV exported from a spreadsheet with columns id, name, pmin, pmax, a, b, c.

Interpreting results

  • System lambda: the marginal cost of serving one more MW — the incremental cost shared by all units operating between their limits.
  • Status: a unit "At max" has no upward reserve; a unit "At min" is too expensive to load further; "Marginal" units set the system lambda.
  • Savings vs. naive: how much the economic schedule beats loading every unit proportionally to capacity.

Limitations

  • Single-period dispatch — no ramp limits, start-up costs, or unit commitment.
  • Losses are a flat percentage; there is no network B-matrix or transmission line limit.
  • Real-power only — no reactive dispatch or voltage optimisation. For network constraints, pair with the Load Flow and Contingency studies.
