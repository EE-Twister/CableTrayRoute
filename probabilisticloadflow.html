Probabilistic / Monte Carlo Load Flow

Method & Assumptions

The probabilistic load flow runs the AC Newton-Raphson solver across many randomly sampled scenarios. Each scenario draws a system load multiplier and a generation multiplier from the distributions you define, then solves a single load flow. Aggregating the results produces probability statistics for bus voltages and system losses.

Per-scenario model:

loadScale ~ loadDist     Pd = Pd_base × loadScale
genScale  ~ genDist      Pg = Pg_base × genScale

The base case is taken from the current project's one-line diagram (draw it on the One-Line page). A seeded random number generator makes every run fully reproducible: the same seed always yields identical results.

Distributions supported: constant, normal (mean, sd), uniform (min, max), triangular (min, mode, max), beta (α, β on [0,1] — ideal for renewable capacity factor), and empirical (bootstrap from a sample list). Optional clamp bounds keep multipliers physical (e.g. load ≥ 0).

Voltage limits (ANSI C84.1 Range A): a scenario is counted as a violation if any bus falls outside 0.95–1.05 pu. The probability of violation is reported overall and per bus.

References: IEEE Std 399-1997 §14; Borkowska (1974), "Probabilistic Load Flow", IEEE Trans. PAS-93.

Probabilistic / Monte Carlo Load Flow Help

This study quantifies how uncertain load and renewable generation affect bus voltages and system losses by running thousands of randomized load-flow scenarios.

When to use

  • Renewable-rich or EV-rich feeders where a single deterministic load flow is not representative.
  • Hosting-capacity studies — what is the probability that PV back-feed pushes a bus above 1.05 pu?
  • Risk-based planning — report P(undervoltage) instead of a single worst case.
  • Loss estimation with confidence bands (P5–P95) rather than a point value.

Choosing distributions

  • Normal — symmetric load uncertainty around an expected multiplier.
  • Beta (α, β) — bounded on [0, 1]; the standard choice for renewable capacity factor (e.g. α=2, β=5 skews toward lower output).
  • Triangular — when only a min, most-likely, and max are known.
  • Empirical — bootstrap directly from measured meter or SCADA samples.

Interpreting results

  • P(voltage violation) — fraction of converged scenarios with any bus outside 0.95–1.05 pu.
  • Histograms — the probability distribution of system loss and minimum bus voltage; the dashed line marks the voltage limit.
  • Per-bus risk — the probability that each bus violates its under- or over-voltage limit.

Limitations

  • Uniform scaling — all loads share one multiplier; all generators share another. For per-bus profiles, provide a custom model via the API.
  • No correlation between the load and generation distributions.
  • Diverged scenarios are excluded from the statistics and reported in the warnings.
