Design Rule Checker

Method & Checks Performed

The Design Rule Checker (DRC) validates your routing project against NEC and IEEE requirements after routing has been completed in the Optimal Route tool. Run the DRC after every routing session to catch code violations early.

DRC-01 — Tray Fill (NEC 392.22(A))
Flags any tray whose cable fill exceeds the configured limit (default 40 %). Reports a warning when fill is within 10 % of the limit.
DRC-02 — Voltage Segregation (NEC 392.6(H))
Detects tray segments containing cables from different cable groups (e.g., HV and LV cables sharing the same tray without a physical barrier).
DRC-03 — Ampacity Derating (NEC 310.15)
Applies bundling derating factors based on the number of current-carrying conductors in each tray section and flags cables whose design current exceeds the derated ampacity.
DRC-04 — Grounding Conductor (NEC 250.122)
Warns when a power cable record does not include a grounding (EGC) conductor size.
DRC-05 — Unrouted Cables
Lists cables present in the Cable Schedule that have no assigned tray or field route after routing.

Run Options

NEC 392.22(A) default is 40 %.

Design Rule Checker Help

The DRC reads your Cable Schedule and Raceway Schedule from the project store, then applies a set of NEC and IEEE rules to identify potential code violations.

Prerequisites: Complete the Cable Schedule, Raceway Schedule, and Optimal Route before running the DRC so that tray-fill data and routing assignments are available.

Results are colour-coded: red = error (must resolve), amber = warning (review), blue = info (advisory).
